rm -rf 大法

2020.09.17

稍微留意一点新闻的人，都会注意到删库跑路的程序员的新闻，再稍微了解点 shell 相关知识的人都知道想要做的彻底没有退路，很有可能是 rm -rf 大法的力量，因此，网上写这些命令处理自己日常工作的人，都会不厌其烦的写上，使用时一定要慎重。今日偶然发现这个命令在 windows 系统下特别好的一个用法。

事情是这样的，电脑上有一个文件夹，需要删除，但是我关闭了所有与之相关的程序，还是无法删除，提示占用，像我这样对电脑有点强迫症加洁癖的人（我可能是那些所谓的精神洁癖，日常生活中反而不太在意），当然不能忍受一个没用的文件夹在那里，一定要删掉，火绒的粉碎文件多数情况下是可以搞定这个问题的，但是缺点是电脑重启后文件夹才能消失，这我哪里能忍啊，今天突然灵机一动， rm -rf 是不是可以搞定？对于这种小事情，当然要做行动派，于是打开我的 wsl 安装的 Ubuntu，进入命令行：

cd /mnt/d
rm -rf xxx  #xxx 代表我想干掉的文件夹
ls -a

瞬间失去了那个文件夹的踪迹，这说明，以后有删不掉的流氓软件，其实很好搞啊，程序员都可以被搞得亡命天涯，还有程序员都怕的命令搞不定的文件/文件夹？从此再也不用怕这些删不掉的文件了。

友情提示，文件名要检查八百遍，最好别用通配符，用的时候也要确保你输入的部分没重复，不然…… 不跑路也有的哭。

真是一物降一物，不能因为 rm -rf 威力大就恐惧他（我一般都是有图形界面都鼠标搞掉，怕出事，现在应该反思一下，本质上还是自信心不足吧），用好了真是无坚不摧。
